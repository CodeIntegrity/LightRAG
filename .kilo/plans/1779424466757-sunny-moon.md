## 修复方案

### Phase 1: 后端严重缺陷 (B1-B5)

#### B1: `_build_runtime_global_config` → `_build_global_config`

**文件**: `lightrag/lightrag.py`
**操作**: `arebuild_all_custom_chunks_graphs` 方法中，将 `global_config=self._build_runtime_global_config()` 改为 `global_config=self._build_global_config()`。需验证两个方法返回值差异：
- 旧方法返回 `asdict(self)` + (已退役的) `active_prompt_groups`
- 新方法返回 `asdict(self)` + `addon_params` + `role_llm_funcs` + `llm_cache_identities`
- 需确认 `merge_nodes_and_edges` 可兼容新格式，或需添加适配层

#### B2: `document_loading_engine` 未定义

**文件**: `lightrag/api/config.py` + `lightrag/api/routers/document_routes.py`
**操作**: 
1. 在 `config.py` 中添加 `document_loading_engine` 参数，从环境变量 `DOCUMENT_LOADING_ENGINE` 读取，默认值 `"DOCLING"`
2. 或：将 `document_routes.py` 中 `global_args.document_loading_engine == "DOCLING"` 替换为环境变量直读 `os.getenv("DOCUMENT_LOADING_ENGINE", "DOCLING")`

#### B3: `ainsert_custom_chunks` 签名不匹配

**文件**: `lightrag/api/routers/document_routes.py` + `lightrag/lightrag.py`
**操作**:
1. 在 `lightrag.py` 的 `ainsert_custom_chunks` 签名中添加 `file_path: str | None = None` 参数
2. 移除 `file_path = normalize_document_file_path("")` 硬编码，改为使用传入参数
3. 恢复 `doc_status` 写入（PROCESSING → PROCESSED/FAILED）和 `merge_nodes_and_edges` 调用
4. 恢复 `_validate_custom_chunks_against_embedding_limit` 验证（若方法仍存在）

#### B4: `ainsert_custom_chunks` 不写 doc_status

**文件**: `lightrag/lightrag.py`
**操作**: 在实体提取完成后，写入 doc_status 记录，设置 `source: "custom_chunks"` 元数据以匹配 `arebuild_all_custom_chunks_graphs` 的过滤逻辑

#### B5: `insert_text`/`insert_texts` 工作区隔离

**文件**: `lightrag/api/routers/document_routes.py`
**操作**: 将重复检测代码中的 `rag.doc_status` 替换为 `current_rag.doc_status`

### Phase 2: 前端缺陷 (F1-F3)
