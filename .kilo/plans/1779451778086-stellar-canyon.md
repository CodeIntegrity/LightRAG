## 修复项

### Bug 1（严重）：`_process_extract_entities` 在无 pipeline_status 时异常处理崩溃

- **位置**：`lightrag/lightrag.py:1593-1599`
- **根因**：`ainsert_custom_chunks` 调用时未传 `pipeline_status` / `pipeline_status_lock`，若 `extract_entities` 失败，except 块访问 `None` 导致 `AttributeError`/`TypeError` 遮盖原始异常。
- **修复**：在 except 块增加 None 守卫，仅当两个参数均非 None 时才更新 pipeline_status。
- **验证**：独立测试该路径（mock `extract_entities` 抛异常 → 确认原始异常不被遮盖、FAILED 状态正常写入）。

### Bug 2（中）：`ainsert_custom_chunks` 错误处理可遮盖原始异常

- **位置**：`lightrag/lightrag.py:1562-1573`
- **根因**：except 块内 `doc_status.upsert(...)` 写 FAILED 状态时自身也可能抛异常，吞噬原始异常。
- **修复**：在 `doc_status.upsert(...)` 外覆 try-except，仅记录日志不阻断 `raise`。
- **验证**：mock `doc_status.upsert` 抛异常 → 确认原始异常仍向上传播。

### Bug 3（中）：`rebuild_custom_chunks_graph` 端点 TOCTOU 竞态

- **位置**：
  - 端点：`lightrag/api/routers/document_routes.py:2931-2941`
  - 核心：`lightrag/lightrag.py:1663-1665`
- **根因**：端点检查 `busy=False` 后通过 `background_tasks.add_task` 调度，两个并发请求可同时通过检查（均返回 200），但实际 bg task 内部检查到 `busy=True` 时会抛 `RuntimeError`，用户不可见。
- **修复**：在端点内原子性地设置 `busy=True`（与检查在同一个 lock 临界区内），bg task 不再重复检查/设置。或者将 `arebuild_all_custom_chunks_graphs` 中的 `RuntimeError("Pipeline is busy")` 改为降级处理（记录日志并直接返回 `skipped` 计数），让两次调度的任务自然串行化。
- **建议方案**：采用端点原子设置 `busy=True` 方案，因为竞态窗口很小且改动最小化。
- **验证**：无简单自动化验证手段（需并发测试），改为代码审查 + 逻辑推演确认。

### Bug 4（低）：`arebuild_all_custom_chunks_graphs` 缩进异常

- **位置**：`lightrag/lightrag.py:1641-1903`（整个方法体）
- **根因**：方法体使用 8 空格缩进，类中其余方法为 4 空格，不符合 PEP 8。
- **修复**：将方法体缩进统一为 4 空格。
- **验证**：`ruff check lightrag/lightrag.py` 通过。

### Bug 5（低）：`del pipeline_status["history_messages"][:]` 含义混淆

- **位置**：`lightrag/lightrag.py:1695`
- **根因**：`del d[k][:]` 语法混淆，实际效果等同 `.clear()`，但容易误读为删除 key。
- **修复**：改为 `pipeline_status["history_messages"].clear()`。
- **验证**：`ruff check` 通过。

## 优化项

### 优化 1：I/O 期间持有 pipeline_status_lock

- **位置**：`lightrag/lightrag.py:1663-1667`
- **问题**：`await self.doc_status.get_docs_by_statuses(...)` 在 `async with pipeline_status_lock` 内调用，DB 查询耗时期间锁被持有，阻塞所有其他使用者。
- **优化**：将 `get_docs_by_statuses` 移到锁外执行，锁内仅做 `busy` 检查 + `busy=True` 设置 + 基于已查询结果的 `custom_docs` 过滤。
- **注意**：竞态：锁外查询结果在重新获取锁时可能已过期。当前场景下（`busy=True` 阻止并发变更），风险可接受。
- **验证**：逻辑审查确认不变性。

### 优化 2：端点预计算 doc_id 冗余

- **位置**：`lightrag/api/routers/document_routes.py:2959-2961`
- **问题**：端点调用 `compute_mdhash_id(sanitize_text_for_encoding(...))` 预算 doc_id，而 `ainsert_custom_chunks` 内部会对同一文本再次 sanitize。
- **优化**：当 `request.doc_id` 为 None 时，传入 `doc_id=None`，让 `ainsert_custom_chunks` 自行计算。保留 `request.doc_id` 非 None 时的显式传入路径。
- **验证**：逻辑审查确认两种情况 doc_key 一致。
