# Goal

在 Prompts 页面新增"通用知识图谱"中文预置提示词，前端内置 preset YAML 模板，支持"从预设加载"按钮一键全量替换编辑器内容。左侧默认展示预设列表（预设可切换），一个工作区只允许激活一套提示词。

# Architecture

- 纯前端方案：预设 YAML 字符串 bundled 在 JS 文件中，零后端改动。
- 左侧边栏改为展示预设列表（替换当前"Files"标题为"预设"），用户点击预设即加载到右侧编辑器。
- **初始加载行为**：当 workspace 无已保存文件时，编辑器自动载入默认预设（"通用知识图谱"）的内容作为草稿模板。这只是编辑态填充，不会自动激活或落盘。用户仍需手动保存 + 启用。
- 编辑器上方新增"新建空白"按钮，支持用户从零开始手写。
- 现有文件列表管理能力保留：用户基于预设编辑 → 保存 → 生成 workspace 文件 → 激活。
- 一个工作区只允许一个 active prompt，激活新文件时自动解除旧文件的 active 状态。

# Preset Content Design

## 预设名称与结构

一个预设："通用知识图谱"（默认），包含三个 section：

| section | 说明 |
|---------|------|
| `entity_types_guidance` | 中文实体类型定义，基于 prompt.py 默认类型翻译扩展 |
| `entity_extraction_examples` | 复用 prompt.py 内置 text 模式 examples（英文，格式参考） |
| `entity_extraction_json_examples` | 复用 prompt.py 内置 JSON 模式 examples（英文，格式参考） |

## entity_types_guidance 内容（中文）

```yaml
entity_types_guidance: |
  请将每个实体归类为以下类型之一。若无匹配类型，归为"其他"。

  - 人物: 真实或虚构的人类个体
  - 生物: 非人类的生命体（动物、神话生物等）
  - 组织: 公司、机构、政府机关、团体
  - 地点: 地理场所（城市、国家、建筑、区域）
  - 事件: 事件、事故、仪式、会议
  - 概念: 抽象思想、理论、原则、信仰
  - 方法: 流程、技术、算法、工作流
  - 内容: 创作或信息作品（书籍、文章、电影、报告）
  - 数据: 定量或结构化信息（统计、数据集、测量值）
  - 人造物: 人类创造的物理或数字对象（工具、软件、设备）
  - 自然物: 自然界的非生命对象（矿物、天体、化合物）
```

# UI Changes

## 左侧边栏

```
当前 (Files):
  ┌─────────────────┐
  │ FILES            │
  │ ┌─────────────┐ │
  │ │ entity-type  │ │
  │ │ v1 · workspace│ │
  │ │  [已启用]     │ │
  │ └─────────────┘ │
  │ ...             │
  └─────────────────┘

改为 (预设):

  ┌─────────────────┐
  │ 预设            │
  │ ┌─────────────┐ │
  │ │ 通用知识图谱  │ │  ← 默认选中（高亮）
  │ │ 中文默认预设  │ │
  │ └─────────────┘ │
  │                 │
  │ 已保存提示词    │  ← 分隔线 + 副标题
  │ ┌─────────────┐ │
  │ │ entity-type  │ │
  │ │ v1 · 已启用  │ │
  │ └─────────────┘ │
  │ ...             │
  └─────────────────┘
```

规则：
- 初始加载时（load 完成），若当前 workspace 无任何已保存文件，自动将默认预设（"通用知识图谱"）的 content 填入编辑器，左侧预设区对应项高亮
- 若 workspace 有文件，按现有 `chooseInitialFile` 逻辑选择 active 文件载入编辑器，左侧预设区不高亮
- 点击预设：选中高亮，editor 载入对应 YAML（若编辑器有未保存变更，弹出 confirm）
- 点击文件：文件被选中（file 列表项高亮），editor 载入文件内容；preset 高亮同时清除——此时表示用户正在查看/编辑"已保存文件"，而非预设草稿
- 下方"已保存提示词"区域显示当前 workspace 已保存的文件
- 如当前 workspace 无已保存文件，hidden 该区域，只显示预设
- 已启用文件在列表中仍显示绿色"已启用"标签

## 编辑器顶部操作栏

在 promptSlug / version / activateOnSave 行上方新增工具栏：

```
[从预设加载 ▼] [新建空白]                    [↻ 刷新]
```

- **从预设加载**：下拉菜单列出所有预设名称，点击后用对应 YAML 全量替换编辑器内容（若编辑器有未保存内容，先弹 confirm 确认框）
- **新建空白**：清空编辑器内容，重置为空白状态

# Risks

- 预设内容中 examples 可能包含英文人名/组织名，与中文 entity_types_guidance 混用，这是预期行为——示例文本来自上游英文内置模板，entity types 按中文定义。
- `hasUnsavedChanges` 需要在 load/select/save/preset load 各环节正确维护，是实现中最容易出 bug 的点。
- `selectedPresetId` 与 `selectedFileName` 的互斥关系需要严格维护：文件/预设/新建空白三个操作都要正确更新这对状态对，防止同时高亮。
- 未来新增预设只需在 `promptPresets.ts` 数组追加，无需改 UI。
