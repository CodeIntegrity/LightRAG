# 计划：为提示词编辑页增加 YAML 语法高亮

## 现状

- `Prompts.tsx` 编辑器区域使用原生 `<Textarea>` 组件（`font-mono` 等宽字体，无语法高亮）
- 项目已有 `react-syntax-highlighter` 但仅用于 Markdown 只读渲染，不适用于编辑器
- 已通过 `useTheme` hook 管理 light/dark 主题

## 方案

使用 **CodeMirror 6** (通过 `@uiw/react-codemirror` React 封装) 替换原生 `<textarea>`，添加 YAML 语言模式和主题支持。

### 选型理由

| 方案 | 体积 | 适用性 |
|------|------|--------|
| CodeMirror 6 | ~200KB gzipped | 轻量，YAML 支持好，React 集成成熟 |
| Monaco Editor | ~5MB+ | 过重，YAML 编辑无需 IDE 级别 |
| textarea + 高亮覆盖层 | 零依赖但脆弱 | 光标同步、编辑体验差 |

### 依赖

```bash
cd lightrag_webui && bun add @uiw/react-codemirror @codemirror/lang-yaml
```

两个包约 200KB gzipped，对 bundle 影响可控。

## 步骤

### 1. 安装 CodeMirror 依赖

两个包：
- `@uiw/react-codemirror` — CodeMirror 6 的 React 组件封装
- `@codemirror/lang-yaml` — YAML 语言模式扩展

### 2. 创建 `YamlEditor` 组件

**文件：** `lightrag_webui/src/components/ui/YamlEditor.tsx`

封装 `@uiw/react-codemirror`，提供：

- 属性：`value`、`onChange`、`placeholder`、`className`
- YAML 语言模式：`yaml()` 扩展
- 编辑器基本配置：行号、折行、缩进辅助等
- 暗/亮主题适配：通过 `useTheme()` 获取当前主题，选择对应 CodeMirror 主题
  - 暗色：`@uiw/codemirror-theme-vscode-dark`（或使用内置的 `tokyo-night-storm`）
  - 亮色：`@uiw/codemirror-theme-github-light`

> 注：`@uiw/react-codemirror` 已自带多个主题，可能无需额外安装主题包。优先使用内置主题。

### 3. 替换 `Prompts.tsx` 中的 `Textarea`

- 将 `Textarea` 替换为 `YamlEditor`
- 移除 `Textarea` 的 import
- `value`/`onChange` 契约保持一致

### 4. 更新 Vite 代码分割

在 `vite.config.ts` 的 `manualChunks` 中新增 `vendor-codemirror` 分组，包含 `@uiw/react-codemirror` 和 `@codemirror/lang-yaml`。

### 5. 验证

- `bun run lint` — ESLint 通过
- `bun test` — 现有测试继续通过
- `bun run build` — 构建成功，bundle 大小合理

## 影响范围

| 文件 | 操作 |
|------|------|
| `lightrag_webui/package.json` | 新增依赖 |
| `lightrag_webui/src/components/ui/YamlEditor.tsx` | **新建** |
| `lightrag_webui/src/pages/Prompts.tsx` | Textarea → YamlEditor |
| `lightrag_webui/vite.config.ts` | 新增 vendor-codemirror chunk |
