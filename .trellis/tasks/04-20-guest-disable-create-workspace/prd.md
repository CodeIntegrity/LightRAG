# 游客禁用创建工作区

## Goal
调整工作区管理功能，未登录游客不能创建工作区。

## Requirements
- 未登录或游客身份打开工作区管理弹窗时，创建工作区按钮必须禁用。
- 页面提示文案应明确要求先登录。
- 已登录非游客用户的创建工作区行为保持不变。

## Acceptance Criteria
- [ ] 游客或未登录状态下，创建按钮不可点击。
- [ ] 游客或未登录状态下，显示登录后才能创建的提示。
- [ ] 管理员或普通登录用户仍可创建工作区。

## Technical Notes
- 变更范围限定在 WebUI 工作区管理组件及其测试。
- 不修改后端接口契约。
